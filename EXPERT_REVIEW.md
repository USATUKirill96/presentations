# Экспертный режим ревью

Этот режим добавляет к основному конвейеру две новые роли:
- `критик` — ищет слабые места, задаёт неудобные, но честные вопросы
- `эксперт` — отвечает на вопросы, проводит веб-исследование и валидирует факты

Обычно его имеет смысл запускать после `03_writer`, когда уже существует
`{PROJECT}/content/slides_content.md`, и до `04_designer`, чтобы не оформлять сырой материал.

## Какие файлы участвуют

### Личности агентов
- `agents/06_critic.md`
- `agents/07_expert.md`
- `agents/08_review_orchestrator.md`

### Артефакты ревью (в директории проекта)
- `{PROJECT}/content/expert_review/critic_questions.md`
- `{PROJECT}/content/expert_review/expert_findings.md`
- `{PROJECT}/content/expert_review/expert_answers.md`
- `{PROJECT}/content/expert_review/final_qa.md`

## Как запускать

### Рекомендуемый способ: оркестратор
Открой новую сессию и напиши:

> Прочитай файл `agents/08_review_orchestrator.md`, затем запусти критика и эксперта параллельно как сабагентов для проекта `projects/{project-name}`. Пусть они обсудят `{PROJECT}/content/slides_content.md`, подготовят вопросы и ответы по содержанию презентации, укажут на неточности при необходимости и соберут итог в `{PROJECT}/content/expert_review/final_qa.md`.

Эта формулировка явно разрешает:
- запускать сабагентов
- работать параллельно
- отправлять эксперта в интернет за факт-чеком

**Важно:** при запуске всегда указывай путь к проекту, чтобы агенты знали с какими файлами работать.

## Что получится на выходе

### Промежуточные файлы
- `critic_questions.md` — список сильных вопросов к материалу
- `expert_findings.md` — независимый факт-чек эксперта
- `expert_answers.md` — ответы эксперта на вопросы критика

### Финальный файл
- `final_qa.md` — единый список вопросов, ответов, неточностей и рекомендуемых правок

## Если хочешь запускать роли по отдельности

### Только критик
> Прочитай файл `agents/06_critic.md` и выполни задачу по `{PROJECT}/content/slides_content.md` для проекта `projects/{project-name}`.

### Только эксперт
> Прочитай файл `agents/07_expert.md`, выполни факт-чек `{PROJECT}/content/slides_content.md` для проекта `projects/{project-name}`, используй интернет и при необходимости запускай сабагентов для поиска и валидации.

## Как использовать результат
1. Открой `{PROJECT}/content/expert_review/final_qa.md`
2. Исправь `{PROJECT}/content/slides_content.md` руками или попроси `03_writer` внести правки
3. При крупных изменениях запусти экспертный режим ещё раз
4. Когда вопросы закрыты, переходи к `04_designer`
