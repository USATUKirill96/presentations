# Бриф презентации

## Тема
Claude Code и AI-assisted development — продвинутые возможности

## Подзаголовок
Инструменты и практики которые вы (скорее всего) ещё не используете

## Целевая аудитория
- **Кто они:** Разработчики, которые уже используют Claude Code в повседневной работе
- **Уровень подготовки:** Средний — знакомы с основами, но не знают о продвинутых возможностях
- **Что они уже знают:** Базовое использование Claude Code, умеют ставить задачи агенту
- **Отношение к теме:** Скептичны к AI-хайпу, нужны конкретные практики, не общие слова
- **Что они хотят получить:** Конкретные инструменты и паттерны для немедленного применения

## Параметры
- **Продолжительность:** 20–25 минут
- **Количество слайдов:** 18–24
- **Основной язык:** русский
- **Дополнительные языки:** английский

## Главная идея
"Ты уже используешь Claude Code — но используешь 20% возможностей. Вот конкретные инструменты и практики, которые изменят твой workflow уже сегодня."

## Области исследования

### Блок A: Ключевые возможности Claude Code (часто игнорируемые)
Для каждого пункта: что это, конкретный пример использования, почему это меняет workflow

1. **CLAUDE.md** — персистентный контекст проекта. Что туда писать, как структурировать.
2. **Hooks** — pre/post хуки на события (tool calls, session start/end). Автоматизация без скриптов.
3. **MCP (Model Context Protocol)** — подключение внешних инструментов (БД, Slack, Jira, браузер).
4. **Slash commands (скиллы)** — кастомные команды для повторяющихся задач.
5. **Параллельные агенты** — запуск нескольких сессий для независимых задач.
6. **Extended thinking** — режим глубокого размышления для сложных архитектурных задач.
7. **Режимы разрешений** — auto-accept, sandbox, настройка доверия инструментам.

### Блок B: Практики и паттерны
1. **Контекст-менеджмент** — /clear, compact, чтение нужных файлов, не всего репо.
2. **Итеративная работа с агентом** — как давать задачи, как декомпозировать.
3. **Ревью кода агентом** — как использовать Claude для code review.
4. **TDD с AI** — сначала тесты, потом код.
5. **Git workflow** — как интегрировать Claude в PR/commit workflow.

### Блок C: Антипаттерны (что делают неправильно)
1. Слишком общие промпты без контекста
2. Гигантские сессии без /clear
3. Не читают что агент сделал — слепо принимают
4. Не используют CLAUDE.md и начинают каждую сессию с нуля
5. Просят сделать всё за один раз вместо итераций

### Блок D: Метрики и примеры
- Конкретные сценарии где Claude Code кратно ускоряет работу
- Примеры задач: рефакторинг, написание тестов, дебаггинг, документация, ревью

## Нарратив и структура
- **Начало (hook):** Боль/проблема — что идёт не так сейчас, почему большинство использует малую часть возможностей
- **Развитие:** Разрыв между тем, что возможно, и тем, что есть. Затем конкретные решения — инструменты и практики (основная часть)
- **Кульминация:** Момент осознания: "я это всё мог использовать уже давно"
- **Финал (CTA):** Конкретные шаги — с чего начать сегодня, упорядоченные по усилию (лёгкое → сложнее)

## Обязательные типы слайдов
- Title slide (название, автор)
- Agenda (план)
- Problem/Hook (1–2 слайда, захват внимания)
- Section dividers (разделители между блоками)
- Content slides (основной контент, 5–6 пунктов максимум на слайде)
- Demo/Example slides (слайды с кодом или конкретными примерами)
- Comparison slides (антипаттерны: ❌ vs ✅)
- Action items (конкретные шаги)
- Q&A / Thank you (финал)

## Голос и стиль
- **Тон:** Уверенный, практичный, без хайпа
- **Язык:** Русский, технический но не занудный
- **Запрещено:** "революционный", "невероятный", "меняет всё", пустые призывы, marketing speak
- **Приветствуется:** Числа, команды, конкретные сценарии, юмор без пафоса, примеры кода

## Дизайн
- **Визуальный стиль:** Технологичный, современный, профессиональный — не слишком корпоративный, не детский
- **Бренд-гайдлайны:** На усмотрение дизайнера (рекомендация: тёмно-синий/антрацит фон для title/divider, светлый для контента, один яркий акцент)
- **Особые пожелания:** Блоки кода с тёмным фоном и моноширинным шрифтом

## Дополнительные требования
- Будь конкретным: примеры команд, реальные сценарии, не абстракции
- Фокус на том что не очевидно новичку в Claude Code
- Demo-слайды должны идти после теоретических — сначала зачем, потом как
- Каждый раздел должен иметь чёткую тему и вывод
- Слайды с кодом: используй реальные примеры команд/конфигов Claude Code
