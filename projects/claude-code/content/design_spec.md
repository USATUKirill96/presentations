# Дизайн-спецификация презентации

## Концепция

Визуальный язык «Code Editor Aesthetics» — тёмная глубокая база для структурных слайдов (title, section dividers), чистый светлый фон для контентных. Единый яркий акцент электрик-циан создаёт технологичность и энергию, ассоциируясь с терминалом и IDE. Общее ощущение — как открыть современный code editor: сфокусированный, спокойный, но с точечными вспышками цвета. Минимум декора, максимум типографики и whitespace.

### Правила применения цветов

- На тёмном фоне заголовки — accent (#00D4FF), body-текст — text-on-dark (#E8EDF2).
- На светлом фоне заголовки — text-on-light (#1A2332), accent используется для маркеров и акцентных линий.
- Блоки кода всегда на code-bg (#0B1222) независимо от фона слайда.
- Цвета positive/negative используются ТОЛЬКО на слайдах с comparison и антипаттернами.
- accent-warm (#FF6B35) — только для ❌ маркеров и предупреждений, не для заголовков.

### Правила типографики

- Межстрочный интервал body-текста: 1.5x размера шрифта (36pt для 24pt текста).
- Межстрочный интервал буллетов: 1.5x (36pt для 24pt текста).
- Расстояние между буллет-пунктами (paraSpaceAfter): 20pt.
- Выравнивание заголовков: по левому краю (кроме title и section — по центру).
- Выравнивание body: по левому краю, без justify.
- Высота блока заголовка на content-слайдах: 1.1" (чтобы длинные заголовки не наезжали на контент).
- Минимальный зазор заголовок → контент: 0.25" (после акцентной линии).

## Геометрия слайда

- Размер: 16:9 (33.867 cm × 19.05 cm / 13.333" × 7.5")
- Поля: 80px (≈2.12 cm) со всех сторон
- Рабочая область: 1760 × 920 px
- Отступ заголовка от верхнего поля: 60px
- Отступ контента от нижнего края заголовка: 40px
- Нижняя зона (для сносок): последние 60px рабочей области
- Колонки (для 2-col лэйаутов): 2 × 820px с зазором 120px между ними

## Лэйауты

### layout: title

Полноэкранный тёмный слайд — первое впечатление.

- **Фон:** Сплошная заливка bg-dark (#0F1B2D). Тонкий линейный градиент снизу-слева (bg-dark) вправо-вверх (bg-dark-secondary #162236, opacity 40%) — для глубины.
- **Заголовок:** Центрирован по горизонтали. Вертикальная позиция — 38% высоты слайда. Inter 56pt, ExtraBold (800), цвет accent (#00D4FF). Максимальная ширина: 80% слайда.
- **Акцентная линия:** Горизонтальная, 120px шириной, 3px высотой, accent (#00D4FF). Расположена по центру, на 24px ниже заголовка.
- **Подзаголовок:** Центрирован, на 24px ниже акцентной линии. Inter 26pt, Regular (400), цвет text-on-dark (#E8EDF2). Максимальная ширина: 70% слайда.
- **Имя докладчика + дата:** Нижний правый угол, 80px от правого края, 60px от нижнего края. Inter 18pt, Regular, цвет text-muted (#7A8BA0). Формат: «{{SPEAKER_NAME}} · {{DATE}}».
- **Декор:** Нет дополнительных элементов. Сила — в пустоте и типографике.

### layout: agenda

Нумерованный список пунктов программы — чистый и сканируемый.

- **Фон:** bg-light (#F0F4F8), сплошная заливка.
- **Заголовок:** Left-aligned, x=80px, y=60px. Inter 40pt Bold (700), text-on-light (#1A2332).
- **Акцентная линия:** Под заголовком, 80px шириной, 3px, accent (#00D4FF). x=80px, y = нижний край заголовка + 12px.
- **Элементы списка:** Начинаются с y = заголовок + 80px. Каждый элемент — строка из двух частей:
  - Номер: Inter 32pt Bold, цвет accent (#00D4FF), фиксированная ширина 60px, x=100px.
  - Текст: Inter 24pt Regular, text-on-light (#1A2332), x=170px, ширина до 1600px.
  - Вертикальный шаг между элементами: 64px.
- **Визуальный разделитель:** Тонкая горизонтальная линия (1px, #D0D8E0) между элементами — опционально.

### layout: content

Рабочая лошадка — заголовок + буллеты. Большинство информационных слайдов.

- **Фон:** bg-light (#F0F4F8), сплошная заливка.
- **Заголовок:** Left-aligned, x=80px, y=60px. Inter 40pt Bold (700), text-on-light (#1A2332). Высота блока: 1.1" (допускает двухстрочные заголовки).
- **Акцентная линия:** Под заголовком, 80px шириной, 3px, accent (#00D4FF). x=80px, y = нижний край заголовка + 12px.
- **Буллет-зона:** Начинается с y = заголовок + 80px, x=80px, ширина 1760px.
  - Маркер: символ «—» (em dash), Inter 24pt Bold, цвет accent (#00D4FF), x=100px.
  - Текст буллета: Inter 24pt Regular, text-on-light (#1A2332), x=140px, ширина до 1640px.
  - Вертикальный шаг между буллетами: 52px (paraSpaceAfter: 20pt).
- **Сноски (если есть):** Привязаны к нижнему краю рабочей области. Inter 12pt, text-muted (#7A8BA0), x=80px, y=980px.

### layout: code

Слайды с примерами кода — визуальный контраст кодового блока на светлом фоне.

- **Фон:** bg-light (#F0F4F8), сплошная заливка.
- **Заголовок:** Left-aligned, x=80px, y=60px. Inter 40pt Bold (700), text-on-light (#1A2332).
- **Акцентная линия:** Под заголовком, как в content.
- **Описание (опционально):** Если есть — Inter 20pt Regular, text-muted (#7A8BA0), x=80px, y = заголовок + 60px. Одна строка.
- **Блок кода:** Прямоугольник со скруглением 8px.
  - Фон: code-bg (#0B1222).
  - Рамка: 1px, #1E3048.
  - Padding внутри: 30px.
  - Позиция: x=80px, y = после описания + 20px (или после заголовка + 80px если нет описания).
  - Ширина: 1760px (вся рабочая область).
  - Высота: авто, но не более 520px (чтобы осталось место для аннотаций).
  - Текст: JetBrains Mono 16pt Regular, code-text (#A8F0D4).
  - Комментарии в коде: text-muted (#7A8BA0).
  - Левый цветной бордюр: 4px, accent (#00D4FF) — для визуального «якоря».
- **Аннотации (НА ЧТО ОБРАТИТЬ ВНИМАНИЕ):** Под блоком кода, формат буллетов как в content, но с маркерами «→» вместо «—». Inter 20pt Regular.
- **Если два блока кода на слайде:** Располагать горизонтально в два столбца (каждый 840px шириной с зазором 80px) ИЛИ вертикально (каждый до 240px высотой). Решает builder на основе объёма кода.

### layout: section

Полноэкранный divider между логическими частями. Минималистичный, даёт «перевести дыхание».

- **Фон:** Сплошная заливка bg-dark (#0F1B2D).
- **Заголовок:** Центрирован по горизонтали и вертикали (немного выше центра — 42% высоты). Inter 52pt ExtraBold (800), accent (#00D4FF). Максимальная ширина: 75% слайда.
- **Акцентная линия:** Горизонтальная, 80px шириной, 3px, accent (#00D4FF). Центрирована, на 20px ниже заголовка.
- **Тизер/подзаголовок:** Центрирован, на 20px ниже линии. Inter 24pt Regular (400), text-on-dark (#E8EDF2). Максимальная ширина: 65% слайда. Italics.
- **Декор:** Никаких дополнительных элементов. Только текст, линия и пространство.

### layout: comparison

Два столбца с визуальным контрастом — для антипаттернов (❌ vs ✅).

- **Фон:** bg-light (#F0F4F8), сплошная заливка.
- **Заголовок:** Left-aligned, как в content.
- **Акцентная линия:** Как в content.
- **Два столбца:** Начинаются с y = заголовок + 80px.
  - **Левый столбец (❌ / «Плохо»):** x=80px, ширина=820px.
    - Заголовок столбца: «❌» + текст, Inter 24pt Bold, negative (#F87171).
    - Подложка: прямоугольник со скруглением 8px, fill #FEF2F2 (очень бледный красный), border 1px #FECACA.
    - Padding внутри: 24px.
    - Текст буллетов: Inter 20pt Regular, text-on-light (#1A2332).
  - **Правый столбец (✅ / «Хорошо»):** x=980px, ширина=820px.
    - Заголовок столбца: «✅» + текст, Inter 24pt Bold, positive (#34D399).
    - Подложка: прямоугольник со скруглением 8px, fill #ECFDF5 (очень бледный зелёный), border 1px #A7F3D0.
    - Padding внутри: 24px.
    - Текст буллетов: Inter 20pt Regular, text-on-light (#1A2332).
- **Разделитель:** Вертикальная пунктирная линия 1px, #D0D8E0, x=940px, от верха столбцов до низа.

### layout: action-items

Нумерованный список действий с цветовой кодировкой сложности.

- **Фон:** bg-light (#F0F4F8), сплошная заливка.
- **Заголовок:** Left-aligned, как в content.
- **Акцентная линия:** Как в content.
- **Элементы:** Начинаются с y = заголовок + 80px. Каждый элемент — карточка:
  - Подложка: прямоугольник со скруглением 8px, fill #FFFFFF, тень не нужна (pptx плохо поддерживает), border 1px #E2E8F0.
  - Ширина: 1760px, высота: авто (~80px).
  - Внутри карточки, слева:
    - Цветной индикатор сложности: круг 16px диаметром, цвет по уровню (см. ниже).
    - Текст действия: Inter 22pt Regular, text-on-light (#1A2332), x = после индикатора + 16px.
    - Оценка времени: справа, Inter 18pt, text-muted (#7A8BA0).
  - Вертикальный шаг между карточками: 16px.
  - **Цвета индикаторов сложности:**
    - Зелёный (#34D399): быстро, 5–15 минут.
    - Жёлтый (#FBBF24): средне, 30 минут.
    - Оранжево-красный (#F87171): существенные усилия, 1+ час.

### layout: finale

Завершающий слайд — Q&A, контакты.

- **Фон:** Сплошная заливка bg-dark (#0F1B2D).
- **Заголовок «Вопросы?»:** Центрирован, вертикальная позиция — 40% высоты. Inter 64pt ExtraBold (800), accent (#00D4FF).
- **Подзаголовок:** Центрирован, на 30px ниже. Inter 26pt Regular, text-on-dark (#E8EDF2).
- **Зона для QR-кода:** Центрирована, на 40px ниже подзаголовка. Плейсхолдер-квадрат 180×180px, border 2px accent (#00D4FF), rounded 8px. Builder может вставить реальный QR или оставить плейсхолдер.
- **Декор:** Как на title-слайде — никаких лишних элементов.

## Декоративные элементы

### Акцентные линии
Единственный декоративный приём — тонкие горизонтальные линии accent-цвета (#00D4FF) под заголовками. Ширина: 80–120px, толщина: 3px. Создают ритм и визуальную привязку, не перегружая. Используются на ВСЕХ слайдах.

### Иконки
Не используются. Презентация строится на типографике и цвете. Эмодзи из контента (❌, ✅, 🟢, 🟡, 🔴) отрисовываются как текст, не как изображения.

### Паттерны и градиенты
- Тёмные слайды: тонкий линейный градиент bg-dark → bg-dark-secondary (40% opacity), direction: 135° (из верхнего-левого в нижний-правый). Создаёт ощущение глубины.
- Светлые слайды: сплошная заливка, без градиентов.

### Разделители
- Между буллетами: не нужны (достаточно whitespace).
- Между столбцами (comparison): вертикальная пунктирная линия 1px, #D0D8E0.
- Между блоками кода (если два на слайде): 30px вертикальный gap.

### Маркеры буллетов
- На content-слайдах: em dash «—» в цвете accent.
- На agenda: числа в цвете accent.
- На action-items: цветные круги-индикаторы.
- На comparison: ❌ / ✅ символы в цветах negative / positive.

## Специальные указания для builder-агента

### Шрифты
- **Inter** — основной. Если недоступен в системе, fallback: Calibri (есть в любой Windows/macOS). Builder должен использовать Inter как primary и НЕ задавать fallback вручную — PowerPoint сделает это сам.
- **JetBrains Mono** — для кода. Fallback: Consolas (Windows) / Menlo (macOS). Если builder не может гарантировать JetBrains Mono — использовать Consolas как safe default.

### Блоки кода
- Реализовывать как shape (rectangle) с заливкой code-bg и текстом внутри.
- НЕ использовать таблицы для кода.
- Левый цветной бордюр (4px accent) — реализовать как отдельный узкий rectangle, примыкающий к левому краю блока.
- Сохранять моноширинную разметку: все пробелы и отступы значимы.

### Слайды с двумя блоками кода (слайды 9, 14)
- Слайд 9: два блока ГОРИЗОНТАЛЬНО (рядом). Левый — JSON (Claude Code), правый — MDC (Cursor). Каждый шириной 840px с зазором 80px.
- Слайд 14: два блока ВЕРТИКАЛЬНО (друг под другом). Верхний — промпты TDD, нижний — план-файл. Уменьшить font до 14pt если не помещается.

### Слайд 19 (comparison)
- Реализовать как два прямоугольника-подложки рядом.
- Левый: бледно-красный фон (#FEF2F2), правый: бледно-зелёный (#ECFDF5).
- Каждый элемент (❌/✅) — одна строка с маркером и текстом.
- Маркеры ❌ и ✅ — как текст, шрифтом, НЕ как изображения.

### Слайд 21 (action-items)
- Цветные круги индикаторов: реализовать как маленькие oval shapes (16×16 px) с заливкой нужного цвета.
- Текст времени справа — right-aligned textbox.
- Между карточками: 16px gap.

### Speaker notes
- Добавлять в notes-поле каждого слайда. Формат: plain text, без markdown-разметки.
- Шрифт notes задаётся PowerPoint автоматически, builder не должен его настраивать.

### Сноски
- Размещать в нижнем-левом углу слайда: x=80px, y=1000px.
- Формат: «¹ Источник, Название, Год» — каждая сноска на новой строке.
- Inter 14pt Light, text-muted (#7A8BA0).

### Переходы между слайдами
- Не добавлять анимации и transitions. Презентация должна быть статичной — анимации ненадёжны при показе на чужом оборудовании.

### Общие правила
- Все размеры в pt для шрифтов, в px для позиционирования (1px = 1/96 дюйма = 9525 EMU).
- Все цвета — HEX без alpha, кроме градиентов где указана opacity.
- Aspect ratio: 16:9. НЕ менять на 4:3.
- Кодировка файла: UTF-8.
