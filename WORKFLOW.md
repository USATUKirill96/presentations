# Конвейер создания презентации

Каждый агент — отдельная сессия. Запускай агентов по порядку,
проверяй артефакт после каждого шага, правь при необходимости, затем запускай следующего.

**Конвейер поддерживает несколько проектов.** Каждая презентация живёт в отдельной
директории внутри `projects/`. Все агенты параметризованы — они читают тему,
аудиторию, стиль и прочие входные данные из `brief.md` проекта.

**Есть два способа запуска:**
1. **Пошагово** — запускаешь каждого агента отдельной сессией (описано ниже)
2. **Одним промптом** — мета-оркестратор `agents/00_pipeline.md` запускает весь конвейер за тебя

---

## Быстрый старт (новый проект с нуля)

Если хочешь создать новую презентацию и не хочешь заполнять бриф руками —
запусти инициализатор. Он задаст тебе 3-5 вопросов, создаст проект и предложит команду для запуска:

> "Прочитай файл agents/00_init.md и помоги создать новый проект презентации"

После этого у тебя будет готовый `projects/{name}/brief.md` и команды для конвейера.

---

## Архитектура конвейера

```
{PROJECT}/brief.md   ←── входной файл, задаёт всё

[01_researcher] → {PROJECT}/content/research.md
                        ↓
               [02_architect] → {PROJECT}/content/outline.md
                                      ↓
                             [03_writer] → {PROJECT}/content/slides_content.md
                                                  ↓
                              [08_review_orchestrator]
                                ↙              ↘
                    [06_critic]                [07_expert]
                                ↘              ↙
                     {PROJECT}/content/expert_review/final_qa.md
                                                  ↓
                                    [09_translator] → {PROJECT}/content/slides_content_{LANG}.md
                                                  ↓
                                        [04_designer] → {PROJECT}/content/design_spec.md
                                                              ↓
                                               [05_builder] → {PROJECT}/output/presentation.pptx
                                                              → {PROJECT}/output/presentation_{LANG}.pptx
```

---

## Структура директорий

```
agents/              ← универсальные инструкции агентов (не трогать)
templates/           ← шаблон брифа для новых проектов
projects/            ← один подкаталог на каждую презентацию
  └── {project-name}/
      ├── brief.md        ← бриф: тема, аудитория, стиль, исследование
      ├── content/        ← артефакты конвейера (твоя зона ревью)
      │   ├── research.md
      │   ├── outline.md
      │   ├── slides_content.md
      │   ├── slides_content_{LANG}.md
      │   ├── design_spec.md
      │   └── expert_review/
      └── output/         ← финальные презентации
          ├── presentation.pptx
          └── presentation_{LANG}.pptx
```

---

## Как начать новый проект

1. Создай директорию: `projects/{project-name}/`
2. Скопируй шаблон: `cp templates/brief_template.md projects/{project-name}/brief.md`
3. Заполни `brief.md` — это единственный файл, который ты пишешь руками
4. Создай поддиректории: `mkdir -p projects/{project-name}/content/expert_review projects/{project-name}/output`
5. Запускай конвейер с шага 1

---

## Шаги

В каждом шаге при запуске агента **указывай проект**. Формат команды:

> "Прочитай файл agents/0X_agent.md и выполни задачу для проекта projects/{project-name}"

---

### Шаг 1 — Исследователь
**Файл агента:** `agents/01_researcher.md`
**Запусти новую сессию и скажи:**
> "Прочитай файл agents/01_researcher.md и выполни задачу для проекта projects/{project-name}"

**Ревью:** Открой `projects/{project-name}/content/research.md`, добавь/убери темы, факты, акценты.
**Готово когда:** Ты доволен набором тем и ключевых мыслей.

---

### Шаг 2 — Архитектор структуры
**Файл агента:** `agents/02_architect.md`
**Запусти новую сессию и скажи:**
> "Прочитай файл agents/02_architect.md и выполни задачу для проекта projects/{project-name}"

**Ревью:** Открой `projects/{project-name}/content/outline.md`. Проверь порядок слайдов, нарратив,
нет ли лишних/недостающих слайдов.
**Готово когда:** Структура и поток истории тебя устраивают.

---

### Шаг 3 — Автор контента
**Файл агента:** `agents/03_writer.md`
**Запусти новую сессию и скажи:**
> "Прочитай файл agents/03_writer.md и выполни задачу для проекта projects/{project-name}"

**Ревью:** Открой `projects/{project-name}/content/slides_content.md`. Правь тексты, заголовки,
примеры кода, заметки спикера под свой стиль.
**Готово когда:** Все тексты финальные.

---

### Шаг 3.5 — Экспертное ревью контента
**Файлы агентов:**
- `agents/06_critic.md`
- `agents/07_expert.md`
- `agents/08_review_orchestrator.md`

**Рекомендуемый запуск в новой сессии:**
> "Прочитай файл agents/08_review_orchestrator.md, затем запусти критика и эксперта параллельно как сабагентов для проекта projects/{project-name}. Пусть они обсудят контент, подготовят вопросы и ответы, укажут на неточности и соберут итог в projects/{project-name}/content/expert_review/final_qa.md"

**Что смотреть после запуска:**
- `projects/{project-name}/content/expert_review/final_qa.md` — сводный список вопросов, ответов и правок
- `projects/{project-name}/content/expert_review/critic_questions.md` — если хочешь увидеть только скептическую линию
- `projects/{project-name}/content/expert_review/expert_answers.md` — если хочешь увидеть только ответы и источники

**Готово когда:** Основные вопросы закрыты, спорные формулировки ослаблены или подтверждены.

Подробная инструкция: `EXPERT_REVIEW.md`

---

### Шаг 3.7 — Переводчик (опционально)
**Файл агента:** `agents/09_translator.md`
**Запусти новую сессию и скажи:**
> "Прочитай файл agents/09_translator.md и переведи контент на английский для проекта projects/{project-name}"

Или для другого языка:
> "Прочитай файл agents/09_translator.md и переведи контент на немецкий для проекта projects/{project-name}"

**Ревью:** Открой `projects/{project-name}/content/slides_content_en.md` (или `_de.md` и т.д.).
Проверь качество перевода, техническую терминологию, заметки спикера.
**Готово когда:** Перевод звучит естественно и все термины на месте.

**Примечание:** Этот шаг опционален. Если презентация нужна только на одном языке —
пропусти и переходи к дизайнеру. Перевод можно сделать и после сборки,
но удобнее — до дизайна, чтобы проверить контент на всех языках.

---

### Шаг 4 — Дизайнер
**Файл агента:** `agents/04_designer.md`
**Запусти новую сессию и скажи:**
> "Прочитай файл agents/04_designer.md и выполни задачу для проекта projects/{project-name}"

**Ревью:** Открой `projects/{project-name}/content/design_spec.md`. Скорректируй цвета, шрифты,
стили под корпоративный брендинг если нужно.
**Готово когда:** Дизайн-спек согласован.

---

### Шаг 5 — Строитель PPTX
**Файл агента:** `agents/05_builder.md`
**Запусти новую сессию и скажи:**
> "Прочитай файл agents/05_builder.md и выполни задачу для проекта projects/{project-name}"

Для сборки на конкретном языке:
> "Прочитай файл agents/05_builder.md и собери английскую версию для проекта projects/{project-name}"

Для сборки всех языков:
> "Прочитай файл agents/05_builder.md и собери все языковые версии для проекта projects/{project-name}"

**Результат:**
- `projects/{project-name}/output/presentation.pptx` — основная версия
- `projects/{project-name}/output/presentation_en.pptx` — английская версия (если переведено)
- и т.д. для каждого языка

Готовые файлы, PowerPoint не нужен.

---

## Как вносить правки между шагами

- **Мелкие правки текста** → редактируй артефакт руками и сразу запускай следующего агента
- **Структурные правки** → опиши правки агенту в той же сессии: "измени слайд 3 — убери пункт про X, добавь пример Y"
- **Итерация на финальном pptx** → в сессии агента-строителя скажи что нужно изменить, он перестроит файл

## Запуск всего конвейера одним промптом

**Файл агента:** `agents/00_pipeline.md`

Мета-оркестратор запускает весь конвейер от исследования до готового .pptx. Два режима:

### Автопилот (без пауз)
> "Прочитай файл agents/00_pipeline.md и запусти конвейер в режиме автопилот для проекта projects/{project-name}"

Все шаги выполняются подряд. Остановка только при критических проблемах. В конце — готовая презентация.

### Пошаговый (с паузами)
> "Прочитай файл agents/00_pipeline.md и запусти конвейер в пошаговом режиме для проекта projects/{project-name}"

После каждого шага оркестратор показывает сводку и спрашивает "продолжить?". Можно вносить правки между шагами.
